# Security and privacy reports

Please do not open a public issue containing financial records, SMS bodies, account identifiers,
credentials, backup files, or a working exploit.

For security-sensitive reports, use GitHub's private vulnerability reporting feature for this
repository. Include the affected version, reproduction steps, impact, and a minimal sanitized proof of
concept. If private reporting is not available, open a public issue containing no sensitive details and
ask the maintainer to establish a private channel.

WHFIN is still a prototype. Do not treat App Lock as database encryption, and do not use development
builds as the sole copy of financial data.
