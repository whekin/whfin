Личный финанс-трекер (Android native), замена ZenMoney. **Полная спека: [SPEC.md](SPEC.md)** — читать перед работой.

## Ключевые решения (не пересматривать без причины)

- **Двухслойная модель**: SMS = pending-черновик в реальном времени, xlsx-выписка = источник правды,
  реконсиляция по мерчант+дата покупки (НЕ по сумме — FX-платежи в разных валютах)
- Суммы: minor units, Long, signed. Переводы = пара транзакций с transferPeerId, не расход/доход
- Категоризация: словарь Merchant→Category, учится навсегда с первого присвоения
- Автокатегоризация: встроенный GeorgiaMerchantPreset (известные сети/сервисы, exact/prefix/contains)
  + пользовательское обучение с приоритетом над пресетом. OSM можно использовать как build-time источник
  обновлений с атрибуцией, но не тащить полную базу в APK
- Корректировка баланса → системная категория Unaccounted (видна в статистике)
- Люди универсальны: отдельная Person-сущность с необязательной ролью (partner/family/friend/colleague/other),
  роль нужна только для фильтров и не влияет на расчёты
- Операцию можно распределить по долям: на себя, совместно, подарок человеку, долг человеку/возврат долга.
  Категория («ресторан») и получатель пользы («Маша») — независимые измерения. Банковская Transaction
  остаётся одной строкой для совпадения с выпиской, доли хранятся дочерними allocation-записями
- Выписки работают отдельно по каждому банковскому счёту (IBAN из файла). Каждая загрузка сохраняет
  период и статистику. Unmatched SMS/manual внутри подтверждённого периода не удаляются автоматически,
  а показываются внутри результата соответствующей загрузки как «Требуют проверки»
- EN+RU локализация и светлая/тёмная тема — с первого дня, не «потом»
- UI-паттерн ввода: НЕ AlertDialog, а bottom sheet через общий `ui/components/FormSheet`
  (serif-заголовок, скруглённые поля, широкая primary-кнопка, imePadding). AlertDialog
  оставлен только для подтверждения деструктивных действий. Сложные формы (редактор
  транзакции) — отдельным полноэкранным шагом

- Реальные выписки и SMS никогда не добавлять в репозиторий, документацию, previews или golden fixtures.
